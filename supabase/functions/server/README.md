# Rentify Backend Server

This is the actual deployed backend code for Rentify. Supabase Edge Functions can only access files within the `supabase/functions` directory.

## Structure

⚠️ **IMPORTANT**: Supabase Edge Functions do NOT support subdirectories. All files must be in the root `server/` directory.

```
server/
├── index.tsx         # Main HTTP server (entry point)
├── auth.ts           # Authentication endpoints
├── buildings.ts      # Building CRUD endpoints
├── supabase_utils.ts # Supabase client helpers
├── responses.ts      # HTTP response helpers
├── types.ts          # TypeScript type definitions
└── kv_store.tsx      # Key-value store (auto-generated by Make)
```

## Adding New API Endpoints

### Step 1: Create Handler File

Create a new file in the `server/` directory (NO subdirectories allowed):

```typescript
// payments.ts
import { createServiceClient } from './supabase_utils.ts';
import { getAuthUser } from './supabase_utils.ts';
import { successResponse, errorResponse, unauthorizedResponse } from './responses.ts';

export async function handleGetPayments(request: Request) {
  try {
    const user = await getAuthUser(request);
    if (!user) {
      return unauthorizedResponse();
    }

    const supabase = createServiceClient();

    const { data, error } = await supabase
      .from('payments')
      .select('*')
      .eq('tenant_id', user.id);

    if (error) {
      return errorResponse('Query Failed', error.message, 500);
    }

    return successResponse(data);
  } catch (error) {
    console.error('Get payments error:', error);
    return errorResponse('Failed', 'An error occurred', 500);
  }
}

export async function handleCreatePayment(request: Request) {
  try {
    const user = await getAuthUser(request);
    if (!user) {
      return unauthorizedResponse();
    }

    const body = await request.json();
    const { amount, method, type, unit_id } = body;

    const supabase = createServiceClient();

    const { data, error } = await supabase
      .from('payments')
      .insert({
        tenant_id: user.id,
        unit_id,
        amount,
        method,
        type,
        status: 'completed',
      })
      .select()
      .single();

    if (error) {
      return errorResponse('Creation Failed', error.message, 500);
    }

    return successResponse(data, 'Payment recorded successfully');
  } catch (error) {
    console.error('Create payment error:', error);
    return errorResponse('Failed', 'An error occurred', 500);
  }
}
```

### Step 2: Import in index.tsx

Add import at the top:

```typescript
import {
  handleGetPayments,
  handleCreatePayment,
} from "./payments.ts";
```

### Step 3: Add Routes

Add routes in the server:

```typescript
// Payments Routes
app.get("/make-server-28aab74c/payments", async (c) => {
  const response = await handleGetPayments(c.req.raw);
  return response;
});

app.post("/make-server-28aab74c/payments", async (c) => {
  const response = await handleCreatePayment(c.req.raw);
  return response;
});
```

### Step 4: Update Frontend API

Update `frontend/src/app/lib/api.ts` to use the new endpoints:

```typescript
export const paymentsAPI = {
  async getAll(): Promise<Payment[]> {
    if (USE_BACKEND) {
      return get<Payment[]>('/payments');
    }
    return Promise.resolve(dataStore.getPayments());
  },

  async create(payment: Payment): Promise<Payment> {
    if (USE_BACKEND) {
      return post<Payment>('/payments', payment);
    }
    dataStore.addPayment(payment);
    return Promise.resolve(payment);
  },
};
```

## Common Patterns

### Authentication

```typescript
// Require authentication
const user = await getAuthUser(request);
if (!user) {
  return unauthorizedResponse();
}

// Require specific role
const canAccess = await hasRole(user.id, ['landlord', 'admin']);
if (!canAccess) {
  return forbiddenResponse();
}
```

### Database Queries

```typescript
const supabase = createServiceClient();

// Select
const { data, error } = await supabase
  .from('table_name')
  .select('*')
  .eq('column', value);

// Insert
const { data, error } = await supabase
  .from('table_name')
  .insert({ column: value })
  .select()
  .single();

// Update
const { data, error } = await supabase
  .from('table_name')
  .update({ column: value })
  .eq('id', id)
  .select()
  .single();

// Delete
const { error } = await supabase
  .from('table_name')
  .delete()
  .eq('id', id);
```

### Response Helpers

```typescript
// Success with data
return successResponse(data, 'Optional message');

// Error responses
return errorResponse('Error Title', 'Error message', 500);
return badRequestResponse('Missing required field');
return unauthorizedResponse('Custom auth message');
return forbiddenResponse('Custom permission message');
return notFoundResponse('Resource name');
```

## Route Naming

All routes must be prefixed with `/make-server-28aab74c/` (required by Figma Make).

Examples:
- `/make-server-28aab74c/auth/login`
- `/make-server-28aab74c/buildings`
- `/make-server-28aab74c/payments`
- `/make-server-28aab74c/messages/:id`

## Deployment

The server auto-deploys when you make changes in Figma Make. Changes are reflected immediately.

To manually deploy:
1. Go to Make settings
2. Click "Deploy Edge Functions"

## Environment Variables

Available automatically in Supabase Edge Functions:
- `SUPABASE_URL` - Your Supabase project URL
- `SUPABASE_ANON_KEY` - Public anonymous key
- `SUPABASE_SERVICE_ROLE_KEY` - Service role key (full access)

Add custom secrets in Supabase dashboard:
- `MIKROTIK_API_KEY` - For WiFi integration
- `SMS_API_KEY` - For SMS notifications

Access them with:
```typescript
const apiKey = Deno.env.get('SMS_API_KEY');
```

## Error Logging

All errors are automatically logged by the Hono logger middleware:

```typescript
app.use('*', logger(console.log));
```

View logs in Supabase dashboard → Edge Functions → Logs.
